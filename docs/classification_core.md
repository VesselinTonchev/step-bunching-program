# Classification core

## General idea

This document contains the central classification logic of the step-bunching program.

The key distinction is between two major types of step bunching:

- **B1-type**
- **B2-type**

The classification is based on the behavior of the minimal step-step distance inside the bunch as the number of steps in the bunch increases.

## B1-type

For **B1-type**, the minimal step-step distance remains approximately constant when the number of steps in the bunch increases.

This implies the presence of **one characteristic length scale** in the bunching problem.

## B2-type

For **B2-type**, the minimal step-step distance decreases when the number of steps in the bunch increases.

This implies the presence of **two characteristic length scales** in the bunching problem.

## Why this matters

This classification is not only descriptive.

It determines:
- how bunches should be monitored,
- how scaling should be interpreted,
- how models should be grouped,
- and how experiment–theory comparison should be organized.

## Monitoring connection

The classification is closely connected with the use of two monitoring schemes:

- **MS-I**: time evolution of global bunch characteristics
- **MS-II**: size-dependent bunch characteristics

These monitoring schemes should be treated as a central part of the step-bunching program.

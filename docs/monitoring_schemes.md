# Monitoring schemes

## General note

Monitoring schemes are a central part of the step-bunching program.

They provide the operational bridge between:
- simulations,
- scaling analysis,
- classification logic,
- and experiment-theory comparison.

## MS-I

Time-evolution monitoring.

Typical quantities include:
- number of bunches
- average number of steps in a bunch
- bunch width
- terrace width
- globally minimal step-step distance

## MS-II

Size-dependent monitoring.

Typical quantities include:
- bunch width as a function of bunch size
- minimal step-step distance as a function of bunch size
- first and last step-step distances in the bunch

## Role in the program

The monitoring logic should be treated not as an auxiliary tool, but as one of the core methodological achievements of the step-bunching line.
